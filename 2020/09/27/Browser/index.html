浏览器是如何运作的

Luminous' blog
2020-09-27

了解浏览器的工作原理有助于我们写出更优质的代码和提供更好的用户体验

浏览器结构

  1. 用户界面：展示除标签页窗口以外的其他用户界面内容
  2. 浏览器引擎：在用户界面和渲染引擎之间传递数据
    • 数据存储持久层
  3. 渲染引擎：渲染用户请求的页面内容
    • 负责网络请求的网络模块
    • 用于解析和执行 js 的 js 解释器
    • 被称为浏览器的内核

不同浏览器内核

  • IE：Trident
  • Firefox：Gecko
  • Safari：Webkit（开源）
  • Chrome/Opera/Edge：Blink（开源）

多进程浏览器

  • 浏览器进程：负责控制浏览器除标签页外的用户界面，包括地址栏、书签、后退和前进按钮，以及负责与浏览器的其他进程协调工作
  • 网络进程：负责发起接受网络请求
  • GPU 进程：负责整个浏览器界面的渲染
  • 插件进程：负责控制网站使用的所有插件
  • 渲染器进程：控制显示 tab 标签内的所有内容，有可能每个标签页都是一个渲染器进程

渲染器进程的过程

  1. 浏览器通过网络线程请求后获取 HTML 数据，再通过 IPC 管道传给渲染器进程，正式进入渲染流程
  2. 主线程解析 HTML 构造 DOM 数据结构，如果遇到 script 标签，就会停止 DOM Tree 构造，先解析 js 脚本再继续构造 DOM Tree
  3. 主线程解析 CSS 进行样式计算确定每个 DOM 节点的样式
  4. 主线程通过 DOM Tree 和计算好的样式来生成 Layout（布局） Tree，Layout Tree 和 DOM Tree 并不是一一对应的，设置了 display: none 的节点不会出现在 Layout Tree 上，而在伪类中设置的 content 内容不会出现在 DOM Tree 上，但是会出现在 Layout Tree 上，Layout Tree 的节点是和最终显示在屏幕上的样式节点对应的
  5. 主线程遍历 Layout Tree 创建一个绘制记录表记录绘制的顺序
  6. 主线程遍历 Layout Tree 生成 Layer（图层） Tree
  7. 主线程将绘制记录表和 Layer Tree 传递给合成器线程，合成器线程将得到的图层分成更小的图块发送给栅格线程
  8. 栅格线程栅格化每个图块并将它们存在 GPU 内存中，返还给合成器线程称为“draw quads”的图块信息
  9. 根据这些信息合成器线程生成合成器帧，合成器帧通过 IPC 管道传送给浏览器进程
  10. 浏览器进程将合成器帧传送给 GPU 进行渲染

重排

当改变一个元素的尺寸位置属性时，需要重新进行样式计算、布局、绘制以及后面所有流程，这种行为称为重排

重绘

当改变某个元素的颜色属性时，不会重新触发布局但是会触发样式计算和绘制，这种行为称为重绘

requestAnimationFrame()

不断进行重绘重排会不断占用主线程，会和 js 执行抢占主线程，可能会因为 js 的执行阻塞主线程导致动画掉帧，requestAnimationFrame()这个 API 会将主线程的任务分散到每一帧的间隔，从而不影响动画的流程

CSS 中的 transform 动画属性

通过该属性实现的动画不会经过样式计算、布局和绘制，而是直接运行在合成器线程和栅格线程，不会受到主线程 JS 执行的影响，也能节省很多运算时间
