Luminous' blog
2020-10-02

请求报文和响应报文

请求报文

主要内容：

  • 请求方法 get/post 等，包含 url、http 协议
  • 主机号 host
  • Accept-Language/Encoding
  • 包含浏览器信息的用户代理 user-Agent

组成：

  • 请求行：请求方式 method 、请求地址 url 和 HTTP 版本
  • 请求头：浏览器信息 user-Agent、接受的语言格式 Accept-Language 和 想要发送给服务器的信息等
  • 请求主体：发送给服务器的数据（内容）

响应报文

主要内容：

  • http 协议，状态码
  • 服务器 server
  • 内容类型 Content-Type 如 text/html，编码方式 charset 如 utf-8
  • 返回的内容（可视）

组成：

  • 状态行：显示请求是否成功
    • 常见请求状态码：
      1. 200 OK 成功
      2. 301 请求对象永久转移
      3. 304 被缓存
      4. 403 Forbidden 被拒绝
      5. 404 Not Found 页面不存在
      6. 5XX 服务器出现问题
  • 响应头：服务器的一些信息和服务器想要告诉浏览器的一些信息
  • 响应主体：正常用户看见的内容

被提问什么是请求报文和响应报文时该如何回答

  • 什么是请求报文？
    • 请求报文包括请求行、请求头和请求主体，主要包含请求的方法、浏览器的信息和发送的数据等信息
  • 什么是响应报文？
    • 响应报文包括状态行、响应头和相应主体，主要包含请求状态、服务器的一些信息和浏览器中可以直接看到的内容等信息

HTTP 协议

使用请求报文和响应报文进行数据交互这种行为是通过 HTTP 协议来规定完成的

使用回调函数

onreadystatechange 事件

  • 当请求被发送到服务器时，需要执行一些基于响应的任务
  • readyState 属性存有 XMLHttpRequest 的状态信息，每当 readyState 改变时，就会触发 onreadystatechange 事件

readyState
存有 XMLHttpRequest 的状态，从 0 到 4 发生变化。

  • 0: 请求未初始化
  • 1: 服务器连接已建立
  • 2: 请求已接收
  • 3: 请求处理中
  • 4: 请求已完成，且响应已就绪

status
状态码，在之前响应报文中介绍过

当 readyState 等于 4 且状态码为 200 时，表示响应已就绪

当网页上存在多个 Ajax 任务时，应该为创建的 XMLHttpRequest 对象编写一个标准的函数，并为每个 Ajax 任务调用该函数，该函数调用应该包含 URL 以及发生 onreadystatechange 事件时执行的任务（每次调用可能不尽相同）

请求行

方法：open(method,url,async);

  • async 默认 true，异步，false 为同步，用于 Ajax 时必须设置异步，同步不常用

  • get

    • 请求时数据时拼接在 url 中的
    • xxx.php?key1=value1&key2=value2

  • post

    • 请求时数据写在发送请求时的 send(string) 函数中

请求头

  • get

    • setRequestHeader() 设置请求头操作可以省略

  • post

    • 必须要用 setRequestHeader() 来设置 HTTP 头，关于 HTTP 头的设置内容之后再深入学习

服务器响应

需要获得来自服务器的响应，可以用 XMLHttpRequest 对象的 responseText 或 responseXML 属性获取返回的数据。可以操作 DOM 元素来修改页面内容 innerHTML，将返回的数据 responseText 赋给 innerHTML 使得用户能看见请求返回的数据。如果后端文件出现问题，还可以用这个方法打印错误提示。

请求主体发送

方法：send();

  • get
    • 不需要设置什么东西，直接发送即可
  • post
    • 可以将需要发送的数据写在参数中，如 send(“key1=value1&key2=value2”)
