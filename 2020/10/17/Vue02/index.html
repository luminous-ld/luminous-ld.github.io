Vue的一些概念

Luminous' blog
2020-10-17

MVVM

什么是 MVVM？

  • MVC：Model、View、Controller

    • Model：数据模型，用来存储数据
    • View：视图界面，用来展示 UI 界面和响应用户交互
    • Controller：控制器，监听模型数据的改变和控制视图行为、处理用户交互

  • MVVM —— Model-View-ViewModel，相对于 MVC 构架多了一个 ViewModel。由于 Controller 主要用来处理各种逻辑和数据转化，复杂业务逻辑界面的 Controller 非常庞大，维护困难，所以采用一个专门的对象 ViewModel 去管理从 Controller 中分离出来的数据和逻辑处理部分，它是 Model 和 Controller 之间的一座桥梁，大大减少了 Controller 中的代码，变得易于测试和维护，只需要 Controller 和 ViewModel 之间做数据绑定即可。

Vue 中的 MVVM

  • View 视图 ——> DOM
  • Model 数据模型 ——> JS 定义的一些数据或者是从网络请求的数据
  • ViewModel 视图模型 ——> Vue 实例，一方面实现了 Data Binding，将 Model 的改变实时地反应到 View 中，另一方面实现了 DOM Listener，当 DOM 发生一些事件时，可以监听到，并且在需要的时候改变对应的 Data。

Vue 的 options 选项

创建 Vue 实例时，传入了一个对象 options，包含很多选项，详细解析查阅官方文档。

入门掌握以下三项

  1. el：类型 string | HTMLElement，决定之后 Vue 实例会管理哪个 DOM 元素
  2. data：类型 Object | Function，Vue 实例对应的数据对象，组件中的 data 必须是函数
  3. methods：类型 {[key:string]:Function}，定义属于 Vue 的一些方法，可以在其他地方调用，也可以在指令中使用

Vue 的生命周期

Vue 的生命周期简单来说就是从创建一个 Vue 实例对象到销毁的全过程，在生命周期的不同阶段调用不同的钩子函数可以实现组件数据管理和 DOM 渲染。

基本生命周期

Vue 的整个生命周期实际上十分复杂，理解起来略有困难，目前刚入门阶段，我就以一个简单的基本周期，借助在各阶段终端调试打印来简单地理解生命周期大概是怎么一回事。

  1. beforeCreated

输出结果：undefined, undefined, undefined

  1. created

输出结果: undefined, [object Object]: {message: 10}, 10

  1. beforeMount

输出结果：<div id="app">{{ message }}</div>, [object Object]: {message: 10}, 10
（markdown 语法新发现，因为 markdown 语法会自动解析 HTML，在笔记中想要引用标签但不想被解析时可以在标签两头各加一个 ` 符号）

  1. mounted

输出结果：<div id="app">10</div>, [object Object]: {message: 10}, 10

  1. beforeUpdated

这个钩子函数是需要在数据发生变化的时候才会调用，所以我们需要在控制台输入 app.message = 20 来观察变化

输出结果：<div id="app">10</div>, [object Object]: {message: 20}, 20

  1. updated

和上一阶段同理，在控制台输入改变数据观察变化

输出结果：<div id="app">20</div>, [object Object]: {message: 20}, 20

  1. beforeDestroy 和 destroyed

beforeDestroy 在实例销毁之前调用，这一步实例仍然可用；而 destroyed 在实例销毁后调用，调用后实例指示的所有东西都会解绑，所有事件监听器会被移除，所有的子实例也会被销毁。同理需要在控制台输入销毁实例命令来观察结果。

输出结果：<div id="app">10</div>, [object Object]: {message: 10}, 10, <div id="app">10</div>, [object Object]: {message: 10}, 10
此时再在控制台对 message 的值进行修改时，会发现它的修改不能触发视图的渲染了。
